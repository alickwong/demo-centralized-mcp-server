## Implementation Steps

### Phase 1: Prerequisites (manual, ~30 min)

**Step 1.1: Okta Developer Account Setup**
- Create free Okta Developer account at developer.okta.com
- Create OIDC Web Application
- Create 2 groups: `analysts`, `finance-admins`
- Create 2 test users:
  - Alice → assigned to `analysts` group
  - Bob → assigned to `finance-admins` group
- Note: Okta domain, client ID, client secret, issuer URL
- Configure Okta to include `groups` claim in ID token and access token

**Step 1.2: AWS Account**
- Ensure AWS CLI configured with appropriate IAM permissions
- Permissions needed: Bedrock AgentCore, Lambda, ECS, ECR, IAM, CloudWatch
- Choose deployment region (must be AgentCore-supported: us-east-1, us-west-2, etc.)

### Phase 2: Setup Notebook (01_setup.ipynb)

**Cell 1: Dependencies + Config**
```
pip install bedrock-agentcore boto3 strands-agents strands-agents-tools mcp httpx
```
- Load Okta config from environment variables or .env file
- Set AWS region

**Cell 2: Create Weather Lambda**
- Define inline Lambda function: accepts city name, returns mock weather data
- Deploy via boto3 Lambda client
- **Verification:** Invoke Lambda directly, print response

**Cell 3: Create Finance Lambda**
- Define inline Lambda function: accepts query, returns mock revenue/financial data
- Deploy via boto3 Lambda client
- **Verification:** Invoke Lambda directly, print response

**Cell 4: Build + Deploy MCP Server to ECS**
- Build Docker image from `mcp_server/` directory
- Push to ECR
- Create ECS Fargate service with public endpoint
- **Verification:** Health check against ECS endpoint

**Cell 5: Create AgentCore Gateway**
- Create OAuth2 authorizer configured with Okta OIDC:
  - Issuer URL: `https://{okta-domain}/oauth2/default`
  - JWKS URI: `https://{okta-domain}/oauth2/default/v1/keys`
  - Audience: Okta client ID
- Create Gateway with the authorizer
- Register 3 targets:
  - Target 1: Weather Lambda (unrestricted)
  - Target 2: Finance Lambda (restricted)
  - Target 3: MCP Server on ECS (unrestricted)
- **Verification:** List tools via Gateway endpoint (should show all targets)

**Cell 6: Define Cedar Policies**
- Create AgentCore Policy store
- Define policies:
  ```cedar
  // Allow all users to access weather tools
  permit(principal, action, resource)
  when { resource.targetName == "WeatherTools" };

  // Allow all users to access CRM MCP server
  permit(principal, action, resource)
  when { resource.targetName == "CRMServer" };

  // Only finance-admins can access finance tools
  permit(principal, action, resource)
  when {
    resource.targetName == "FinanceTools" &&
    principal.groups.contains("finance-admins")
  };

  // Default deny (implicit in Cedar)
  ```
- **Verification:** Policy verification cell — test policy evaluation with:
  - Alice (groups: ["analysts"]) + FinanceTools → expect DENY
  - Bob (groups: ["finance-admins"]) + FinanceTools → expect ALLOW
  - Alice + WeatherTools → expect ALLOW

**Cell 7: Save Config**
- Write `gateway_config.json` with Gateway URL, target names, Okta config

### Phase 3: MCP Server (mcp_server/)

**Dockerfile:**

### Phase 4: Demo Notebook (02_demo.ipynb)

**Cell 1: Load Config + Pre-Auth Both Users**
- Load `gateway_config.json`
- Authenticate Alice via Okta → get JWT → decode and print claims (show groups: ["analysts"])

- Create 2 MCPClients, one per user:
  ```python

      gateway_url, headers={"Authorization": f"Bearer {alice_jwt}"}
  ))
  bob_client = MCPClient(lambda: streamable_http_client(
      gateway_url, headers={"Authorization": f"Bearer {bob_jwt}"}
  ))
  ```
- Create 2 Strands Agents, one per user

**Cell 2: Show Available Tools**

**Cell 3: Alice Asks for Weather (ALLOWED)**

**Cell 6: Bob Queries CRM MCP Server (ALLOWED)**

## Risk Mitigation

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| Cedar policy syntax wrong | Medium | Policy verification cell in setup (Step 6) |
| Okta token expired during demo | Low | Pre-auth at demo start, tokens last 1 hour |
| ECS cold start delay | Medium | Pre-warm in setup notebook, use Fargate spot for cost |
| AgentCore Gateway quota limits | Low | Use single demo region, minimal concurrent requests |
| Strands MCPClient connection timeout | Low | Set startup_timeout=60 in MCPClient constructor |
| Lambda cold start during demo | Low | Pre-warm with test invocation in setup |
